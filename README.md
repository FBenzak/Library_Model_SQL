## Banco de Dados Relacional de uma Biblioteca (SQL)

Este projeto consiste no desenvolvimento de um **banco de dados relacional para uma biblioteca**, criado como **projeto acadêmico** durante minha graduação em **Gestão de Tecnologia da Informação**.

## Objetivo
Modelar e implementar um banco de dados relacional capaz de gerenciar:
- Clientes (pessoas físicas e jurídicas)
- Livros e editoras
- Pedidos e itens comprados
- Telefones de clientes e editoras

## Tecnologias Utilizadas
- MySQL  
- SQL (DDL e DML)

## Estrutura do Banco de Dados
O banco de dados inclui as seguintes entidades principais:
- Cliente  
- Pessoa_Fisica  
- Pessoa_Juridica  
- Livros  
- Editora  
- Pedido  
- Item_Pedido  

Os relacionamentos foram implementados utilizando **chaves estrangeiras**, garantindo a **integridade dos dados** e a **normalização** do banco.

## Funcionalidades
- Criação de tabelas normalizadas
- Uso de chaves primárias e estrangeiras
- Inserção de dados realistas para testes
- Consultas SQL utilizando **JOIN** para análise de dados

## Library Relational Database (SQL)

This project consists of the development of a **relational database for a library**, created as an **academic project** during my degree in **Information Technology Management**.

## Objective
To model and implement a relational database capable of managing:
- Clients (individuals and companies)
- Books and publishers
- Orders and purchased items
- Client and publisher phone numbers

## Technologies Used
- MySQL  
- SQL (DDL and DML)

## Database Structure
The database includes the following main entities:
- Cliente  
- Pessoa_Fisica  
- Pessoa_Juridica  
- Livros  
- Editora  
- Pedido  
- Item_Pedido  

Relationships were implemented using **foreign keys**, ensuring **data integrity** and **database normalization**.

## Features
- Creation of normalized tables
- Use of primary and foreign keys
- Realistic data insertion for testing
- SQL queries using **JOIN** for data analysis
